  • Stop EMR application
  • Stop ECS service
  • Delete EventBridge schedules

  • Delete S3 Tables (async)
  • Terraform destroy
  • Final verification

Some Non-FAQs

‚ö° Why Bronze Streaming + Silver/Gold Batch?

Bronze streaming achieves ‚â§3-min SLA for real-time ingestion. Batch processing for Silver/Gold is cost-effective and allows comprehensive DQ checks. This hybrid pattern optimizes freshness (Bronze) while minimizing compute costs (Silver/Gold).

üì® MSK Kafka vs Kinesis Data Streams?

Kafka provides replay capability for Bronze reprocessing‚Äîcritical for recovery and debugging. KRaft mode eliminates Zookeeper complexity. Kafka better handles high throughput (500-700 edits/min) with flexible retention and DLQ support.

üîÑ How Does Step Functions Self-Looping Work?

After Gold DQ passes, Step Functions triggers the next cycle automatically. 15-min initial wait ensures Bronze data is ready (3-min batch + buffer). Each cycle runs to completion before next starts‚Äîno overlap, no gaps. EMR cold start (5-8 min) is accounted for in the 15-25 min cycle time.

‚è∞ How Does the 10-min Watermark Handle Late Events?

Bronze streaming uses 10-min watermark‚Äîevents arriving >10 min after event_timestamp are dropped. Gold hourly aggregations use event_hour partition, so late events in correct hour still aggregate. Trade-off: 10-min balance minimizes dropped events while limiting Iceberg snapshot bloat. For production, consider per-event-type watermarks (log events lower latency vs categorize higher latency).

üîÑ How Is Idempotency Ensured Across Restarts?

Deterministic event_id = SHA256(event_timestamp + wiki + title + rev_id) ensures same event always maps to same ID. Bronze MERGE INTO upserts on event_id‚Äîduplicates are overwritten, not duplicated. On Bronze restart (Lambda-triggered), Kafka replay reprocesses same events but idempotency prevents double-counting. Silver deduplicates by event_id across partitions. Audit trail logs merge operations for verification.

üö® Failure Recovery: Bronze vs Batch?

Bronze Streaming: Lambda monitors CloudWatch metrics (no records in 10 min) ‚Üí triggers EMR job restart. Batch Pipeline: Stops on DQ failure ‚Üí manual investigation ‚Üí restart with fresh data. Kafka retention (7 days) enables Bronze replay for backfilling missing periods.
